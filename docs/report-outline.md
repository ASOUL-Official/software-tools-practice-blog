# 实验报告提纲

## 1. 实验目的

通过构建一个静态技术博客，完整实践 Web 开发环境搭建、代码版本控制、Linux 环境配置、远程管理和站点部署流程。

## 2. 实验环境

- Windows 11
- Git
- Node.js
- VS Code
- VirtualBox
- Ubuntu Server 22.04 LTS
- OpenSSH
- Nginx

## 3. 实验过程

### 项目与代码库配置

说明项目目录结构、Git 初始化、提交记录和远程仓库配置。

### Web 运行环境

说明本地静态站点运行方式、页面结构、资源组织和浏览器访问结果。

### Linux 环境配置

说明虚拟机网络、主机名、用户、权限、环境变量和常用配置文件。

### Linux 账户配置

说明普通用户创建、sudo 权限、home 目录、文件权限和用户组管理。

### 远程管理配置

说明 SSH 服务、密钥认证、远程登录、scp/rsync 文件传输。

### Web 站点部署

说明 Nginx 安装、站点目录、配置文件、服务重载和访问验证。

## 4. 实验结果

最终成果为可访问的课程实践博客站点，并保留 Git 提交历史、Linux 配置截图、SSH 登录截图和部署访问截图。

## 5. 知识总结

| 知识点 | 作用 | 本项目体现 |
| --- | --- | --- |
| Git | 版本控制 | 记录项目每个阶段修改 |
| Node.js | 本地运行环境 | 启动本地预览服务 |
| Linux | 服务运行环境 | 承载 SSH 与 Nginx |
| SSH | 远程管理 | 登录虚拟机并传输文件 |
| Nginx | Web 服务器 | 对外提供静态站点访问 |

## 6. 出现问题及解决方案

可记录网络不通、端口占用、权限不足、Nginx 配置错误、Git 推送失败等问题。

## 7. 心得体会

从工具链角度总结软件开发不是单个软件的使用，而是代码、环境、协作、部署和文档共同构成的工程流程。

## 8. 技术展望

后续可将手动部署升级为 GitHub Actions 自动部署，也可将静态站点从手写 HTML 升级为 Hexo、VitePress 或 Astro。
